# Animal battlefield
## 시작하기 전 중요 사항

이 프로젝트는 프로그래밍 실력 향상을 위해 제작되었습니다. 만약의 상황을 대비해서, 프로젝트 파일을 git으로 받아오는 것보다, zip으로 다운로드 후 실행하는 것을 강력히 추천합니다.



animal_battlefield.bat를 실행해서 시작할 수 있습니다. 만약 config.py 파일이 없을 경우, 사용자에게 데이터베이스를 생성하기 위해 필요한 정보를 요구합니다.

만약 데이터베이스 정보를 넘기는 것이 어렵다면, config_example.py에서 빈 문자열을 채우고, 해당 파일의 이름을 config.py로 변경하시면 사용할 수 있습니다.

DB = "(데이터베이스 이름)"
PORT = "(데이터베이스 Port)"
HOST = "(데이터베이스 Host)"
ADMIN_USER = "(데이터베이스 슈퍼계정)"
ADMIN_PASSWORD = "(데이터베이스 슈퍼계정 비밀번호)"


## animal battlefield 소개
**Animal Battlefield**는 Python과 Flask 기반으로 제작된 **데이터베이스 중심 자동 턴제 카드 배틀 게임**입니다.  
이 프로젝트의 핵심은 **전투 중 발생하는 모든 행동과 상태 변화를 세밀하게 데이터베이스에 기록하여, 카드 밸런스 패치 이후에도 과거 전투를 완전하게 재현(replay)** 할 수 있도록 설계된 구조입니다.


## 게임 진행 방식

### 1. 덱 구성
- 덱은 정확히 **4장의 카드**로 구성
- 총 비용(cost)의 합이 **20 이하**여야 ‘완성됨’으로 인정
- 덱 상태:
  - 미완성
  - 완성됨
  - 방어 덱(랜덤 매칭에서 선택될 덱)

### 2. 랜덤 매칭 전투
- 플레이어가 사용할 완성된 덱 선택
- 시스템이 다른 플레이어의 방어 덱 중 하나를 무작위 선택
- 자동 턴제 전투 실행
- 전투 중 모든 카드 상태·행동이 기록됨
- 전투 종료 후 리플레이 시청 가능

### 3. 리플레이
- MatchActions + MatchStates 기반 재현
- 전투 당시의 패치 ID에 맞춘 카드 스탯 재구성
- 정확한 과거 상태로 재생
